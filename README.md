# ASTE Project — Generative T5 Seq2Seq

This project implements Aspect Sentiment Triplet Extraction (ASTE) using a generative sequence-to-sequence approach powered by **T5-small**. It extracts `(Aspect, Opinion, Valence, Arousal)` quadruplets directly from text, avoiding the brittle BIO-tagging and cartesian-product issues of traditional extractive token classifier models.

## Architecture

* **Model Layer:** `google-t5/t5-small` (60M parameters)
* **Input Structure:** `extract aspect opinion sentiment: [Sentence]`
* **Output Structure:** `( aspect | opinion | V | A ) ; ( aspect | opinion | V | A )`
* **Performance:** Reaches **~0.67 F1** (evaluated on exact triplet bounds), vastly outperforming the previous ModernBERT extractive baseline (~0.59 F1) by cleanly handling implicit (NULL) aspects and multi-word span boundaries.

## Project Flow & Quick Start

```bash
# 1. Install dependencies
# (We use transformers, torch, and sentencepiece for the T5 tokenizer)
bash setup.sh
pip install sentencepiece

# 2. Merge the two datasets (Laptop + Restaurant)
python merge_data.py

# 3. Train the model (CPU/GPU)
# The dataloader will automatically linearize JSON objects into the T5 text target format
python train.py \
  --data merged_train.jsonl \
  --epochs 5 \
  --batch_size 8 \
  --grad_accum 2 \
  --lr 3e-4 \
  --encoder google-t5/t5-small

# 4. Predict on the test set
# The generation uses beam search (num_beams=2 or 4) and aggressively parses output text back to JSON
python predict.py \
  --test test_data.jsonl \
  --checkpoint checkpoints/best_model.pt \
  --output predictions.jsonl \
  --encoder google-t5/t5-small
```

## File Structure

```
.
├── setup.sh            # Install core dependencies
├── merge_data.py       # Merge laptop + restaurant → merged_train.jsonl
├── data_loader.py      # T5 tokenizer & triplet linearizer + DataLoader
├── model.py            # ASTEModel (Thin wrapper over T5ForConditionalGeneration)
├── train.py            # Training loop, Triplet-Level F1 Eval, checkpointing
├── predict.py          # Inference via beam search & string parsing
├── report.md           # Model methodology and metrics report
├── merged_train.jsonl  # Combined training data (generated by merge_data.py)
├── test_data.jsonl     # Test data for batch evaluation
├── predictions.jsonl   # Extracted outputs matching target JSONL format
└── checkpoints/
    ├── best_model.pt   # Best checkpoint by val triplet F1
    └── history.json    # Per-epoch metrics
```

## Training Parameters

| Flag | Default | Description |
|---|---|---|
| `--encoder` | google-t5/t5-base | The HuggingFace model string (e.g. `google-t5/t5-small`) |
| `--epochs` | 15 | Number of training epochs |
| `--batch_size` | 8 | Batch size |
| `--grad_accum` | 2 | Gradient accumulation steps |
| `--lr` | 3e-4 | Peak learning rate (Linear decay) |
| `--num_beams` | 4 | Number of beams used during validation decoding |
| `--patience` | 5 | Early stopping patience (by Validation F1) |

## Output Format

The output is written to `predictions.jsonl` matching the task spec:

```json
{"ID": "R001", "Triplet": [{"Aspect": "Indian food", "Opinion": "average to good", "VA": "6.75#6.38"}]}
```
